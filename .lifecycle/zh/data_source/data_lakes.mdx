# 数据湖仓库

<DataLakeIntro />


## 关键理念

- 开放数据格式：支持包括 JSON、Parquet、Avro 在内的多种数据类型，便于存储和处理结构化与非结构化数据。
- 元数据管理：采用共享元数据层，常用如 Iceberg 表格式等来高效组织和管理数据。
- 多样化查询引擎：集成了多种引擎，如增强版的 Presto 和 Spark，以适应不同的分析和人工智能场景。
- 治理与安全：提供强大的数据安全、隐私保护及合规机制，确保数据的完整性和可信赖性。

## 数据湖仓库架构的优势

- 灵活性与可扩展性：无缝管理不同类型的数据，随组织需求进行扩展。
- 成本效益：相比传统方法，提供了更经济的数据存储和处理方案。
- 增强的数据治理：提升了数据的控制、管理和完整性，确保数据处理的可靠性和安全性。
- 人工智能及分析的适配性：非常适合进行复杂的分析任务，包括机器学习和基于人工智能的数据处理。

## StarRocks 的策略

需要考虑的关键因素包括：

- 与目录或元数据服务集成的标准化
- 计算节点的弹性伸缩
- 灵活的缓存机制


## 目录

StarRocks 有内部和外部两种类型的目录。内部目录包含存储在 StarRocks 数据库中的数据元数据。外部目录用于管理存储在外部的数据，包括 Hive、Iceberg、Delta Lake 和 Hudi 管理的数据。页面底部的“更多信息”部分链接到许多其他外部系统。

## 计算节点（CN）扩展

存储与计算的分离简化了扩展过程。由于 StarRocks 计算节点仅存储本地缓存，可以根据负载变化增加或移除节点。

## 数据缓存

计算节点上的数据缓存是可选的。如果您的计算节点因负载模式迅速变化而频繁启动和关闭，或者您的查询通常只涉及最新数据，那么缓存数据可能不是必要的。

## 动手学习

尝试使用基于 Docker 的数据湖仓库。这个[data lakehouse tutorial](../quick_start/iceberg.md)将创建一个基于 Iceberg 和 MinIO 搭建的数据湖，并将 StarRocks 容器作为查询引擎进行连接。加载提供的数据集，或者添加自己的。
