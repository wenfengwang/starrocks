# 显示加载作业

## 描述

此语句用于显示数据库中所有或指定的加载作业信息。只有通过[Broker Load](../data-manipulation/BROKER_LOAD.md)、[INSERT](./INSERT.md)或[Spark Load](../data-manipulation/SPARK_LOAD.md)创建的加载作业才能被显示。您也可以使用`curl`命令查看加载作业信息。从v3.1版本开始，我们推荐您使用[SELECT](../data-manipulation/SELECT.md)语句从`information_schema`数据库的[`loads`](../../../reference/information_schema/loads.md)表中查询Broker Load或Insert作业的结果。更多信息请参考[从HDFS加载数据](../../../loading/hdfs_load.md)、[从云存储加载数据](../../../loading/cloud_storage_load.md)、[使用INSERT加载数据](../../../loading/InsertInto.md)以及[使用Apache Spark™进行批量加载](../../../loading/SparkLoad.md)。

除了上述加载方法，StarRocks还支持使用Stream Load和Routine Load来加载数据。Stream Load是一个同步操作，会直接返回Stream Load作业的信息。Routine Load是一个异步操作，您可以使用[SHOW ROUTINE LOAD](../data-manipulation/SHOW_ROUTINE_LOAD.md)语句来显示Routine Load作业的信息。

## 语法

```SQL
SHOW LOAD [ FROM db_name ]
[
   WHERE [ LABEL { = "label_name" | LIKE "label_matcher" } ]
         [ [AND] STATE = { "PENDING" | "ETL" | "LOADING" | "FINISHED" | "CANCELLED" } ]
]
[ ORDER BY field_name [ ASC | DESC ] ]
[ LIMIT { [offset, ] limit | limit OFFSET offset } ]
```

> **注意**
> 您可以在语句中添加`\G`选项（如`SHOW LOAD WHERE LABEL = "label1"\G;`）来垂直显示输出，而非常规的水平表格格式。更多信息请参考[示例 1](#examples)。

## 参数

|参数|必填|说明|
|---|---|---|
|db_name|否|数据库名称。如果不指定该参数，则默认使用您当前的数据库。|
|LABEL = "label_name"|否|加载作业的标签。|
|LABEL LIKE "label_matcher"|否|如果指定该参数，则返回标签包含label_matcher的加载作业信息。|
|AND|否|如果在WHERE子句中只指定一个过滤条件，则不要指定该关键字。示例：WHERE STATE = "PENDING"。如果在 WHERE 子句中指定两个或三个过滤条件，则必须指定此关键字。示例：WHERE LABEL =“label_name”AND STATE =“PENDING”。 |
|STATE|否|加载作业的状态。根据加载方式的不同，状态也会有所不同。Broker LoadPENDING：加载作业已创建。QUEUEING：加载作业在队列中等待调度。LOADING：加载作业正在运行。PREPARED：事务已提交。FINISHED：加载作业已完成。加载作业成功。已取消：加载作业失败。Spark LoadPENDING：您的 StarRocks 集群正在准备与 ETL 相关的配置，然后将 ETL 作业提交到您的 Apache Spark™ 集群。ETL：您的 Spark 集群正在执行 ETL 作业，然后写入LOADING：HDFS 集群中的数据正在加载到您的 StarRocks 集群，表示加载作业正在运行。PREPARED：事务已提交。FINISHED：加载作业成功。CANCELLED：加载作业正在运行。作业失败。INSERTFINISHED：加载作业成功。CANCELLED：加载作业失败。如果不指定STATE参数，则默认返回所有状态的加载作业信息。如果指定了STATE参数，则仅返回给定状态下的加载作业的信息。例如，STATE = "PENDING" 返回处于 PENDING 状态的加载作业的信息。|
|ORDER BY 字段名称 [ASC \| DESC]|否|如果指定此参数，则输出根据字段按升序或降序排序。支持以下字段：JobId、Label、State、Progress、Type、EtlInfo、TaskInfo、ErrorMsg、CreateTime、EtlStartTime、EtlFinishTime、LoadStartTime、LoadFinishTime、URL 和 JobDetails。要按升序对输出进行排序，请指定 ORDER BY field_name ASC如果要按降序对输出进行排序，请指定 ORDER BY field_name DESC。如果不指定字段和排序顺序，则默认按 JobId 的升序对输出进行排序。|
|LIMIT 限制|否|允许显示的加载作业数量。如果不指定该参数，则显示所有符合过滤条件的加载作业信息。如果指定该参数，例如LIMIT 10，则只返回符合过滤条件的10个负载作业信息。|
|OFFSET offset|否|offset 参数定义要跳过的加载作业的数量。例如，OFFSET 5 会跳过前五个加载作业并返回其余作业。 offset参数的值默认为0。|

## 输出

```SQL
+-------+-------+-------+----------+------+---------+----------+----------+------------+--------------+---------------+---------------+----------------+-----+------------+
| JobId | Label | State | Progress | Type | Priority | EtlInfo | TaskInfo | ErrorMsg | CreateTime | EtlStartTime | EtlFinishTime | LoadStartTime | LoadFinishTime | URL | JobDetails |
+-------+-------+-------+----------+------+---------+----------+----------+------------+--------------+---------------+---------------+----------------+-----+------------+
```

## 使用说明

- SHOW LOAD语句返回的信息自加载作业完成时间（LoadFinishTime）起3天内有效。3天后，信息将无法显示。您可以使用label_keep_max_second参数来修改默认的有效期。

  ```SQL
  ADMIN SET FRONTEND CONFIG ("label_keep_max_second" = "value");
  ```

- 如果LoadStartTime字段的值长时间为N/A，这意味着加载作业严重堆积。我们建议您降低创建加载作业的频率。
- 加载作业消耗的总时间 = LoadFinishTime - CreateTime。
- 加载作业在LOAD阶段消耗的总时间 = LoadFinishTime - LoadStartTime。
