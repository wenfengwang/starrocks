# StarRocksへの貢献

StarRocksへの貢献は、誰からでも熱烈に歓迎されています。StarRocksへの貢献は、コードの貢献に限定されません。以下に、コミュニティへの貢献方法のいくつかをリストします。

| バグの報告                       | StarRocksのバグを報告するために、[issue](https://github.com/StarRocks/starrocks/issues/new/choose)を送信することができます。また、ドキュメントサイトのページ上部の`フィードバック` をクリックしてバグを報告することもできます。 |
| ------------------------------- | ------------------------------------------------------------ |
| **コードの貢献**                | バグの修正や機能の実装など、コードを貢献することができます。 |
| **テストケースの貢献**          | テストケースを貢献することができます。                      |
| **コードのレビューのサポート**  | StarRocksの積極的な貢献者やコミッターの方々は、プルリクエスト(PR)のレビューをサポートすることができます。 |
| **ドキュメントの貢献**          | StarRocksコミュニティは、中国語と英語の両方で大量のドキュメントを保守しています。バグ修正や新しいコンテンツの提案により、ドキュメントの変更に貢献することができます。 |
| **StarRocksユーザのサポート**  | コミュニティ内で困難に直面する新参者のサポートをすることができます。 |
| **StarRocksの宣伝**            | 記事の執筆やプレゼンテーションを行うことにより、私たちの技術を世界に広めるためのサポートをすることができます。 |

> **注記**
>
> ドキュメントへの貢献には、`git commit -s`を使用してコミットに署名してください。それ以外の場合、開発者証明書（DCO）のチェックが失敗し、PRがブロックされる可能性があります。さらに、PRのタイトルに `[Doc]` の接頭辞を付け、**Doc** チェックボックスを選択してください。
> ![ドキュメントのタイトル](../assets/doctitle.png)

## コミュニティのリソース

StarRocksに関するさまざまなヘルプを得る最適な場所は、StarRocksの[Slackチャンネル](https://join.slack.com/t/starrocks/shared_invite/zt-1d0cs3cs2-7eL5Q0cGoBofhyIOx7pLfQ)です。

貢献に関連するディスカッションは、**#contributing-to-starrocks** チャンネルに移動してください。

また、[GitHub](https://github.com/StarRocks/starrocks/)で問題や問題の報告、新機能の提案もすることができます。

## 行動規範

私たちのコミュニティは、厳格に[行動規範](https://github.com/StarRocks/starrocks/blob/main/CODE_OF_CONDUCT.md)に従っています。

## コミュニティの役割

![コミュニティの役割](../assets/contri-1.png)

StarRocksプロジェクトへの参加をお勧めします。新機能、プロジェクトのロードマップ、アーキテクチャに関するディスカッションや、直面する問題の報告など、いろいろな方法で影響を与えることができます。

以下にリストされている役割は、コミュニティへの参加方法の一部です。また、各役割には期待されることも定義されています。

## ユーザーグループ

### 参加者

参加者は、コミュニティに積極的に関与し、StarRocksをより良い状態にするために活動しています。

参加者として、GitHubで問題報告を提出したり、テストケースを貢献したり、ドキュメントの翻訳/修正をしたり、フォーラムやコミュニティでユーザーの質問に回答したり、イベントに参加したり、StarRocksのご利用体験を共有したり、StarRocksにスターを付けたりすることができます。

期待されることと責任:

- コミュニティの[行動規範](https://github.com/StarRocks/starrocks/blob/main/CODE_OF_CONDUCT.md)を守ること。

- ディスカッションやコミュニティイベントに関与すること。

- コミュニティにフィードバックを共有し、他の皆さんに動作の良し悪しを伝えること。

- 改善の提案をすること。

### チャンピオン

StarRocksコミュニティチャンピオンは、StarRocksの技術に精通した情熱的なコミュニティエバンジェリストであり、先駆者でもあります。彼らは、コミュニティでの最新の開発や製品を共有することに喜びを感じ、その情熱はコミュニティの進歩と発展を促進します。

#### ベネフィット

製品

- 新製品/新機能への早期アクセス。

- プロジェクトコアチームとの専用ミーティングへの参加。

コミュニティ

- ウェブサイト上でStarRocksチャンピオンとしてリストアップ。

- 専用の賞品やバッジ。

イベント

- StarRocksイベントへの無料アクセス。

- イベントのための交通費および宿泊費の補填。

- VIPシートとVIPイベント。

#### 要件

1. StarRocks関連のオリジナルかつ品質の高い技術コンテンツ（記事、チュートリアル、ビデオなど）を少なくとも3つ公開していること。
2. スピーカーとしてオンライン/オフラインの技術共有セッションに少なくとも2回参加していること。
3. Slackなどのチャンネルでユーザーの質問に回答していること。

#### その他の要件

1. StarRocksチャンピオンは、StarRocksコミュニティの[行動規範](https://github.com/StarRocks/starrocks/blob/main/CODE_OF_CONDUCT.md)に従う必要があります。
2. StarRocksチャンピオンの資格は1年間有効であり、毎年の貢献度に基づいて毎年再申請する必要があります。

## 開発者グループ

### 貢献者

貢献者はStarRocksのコミッターになることができます。メンバーは、新しい貢献者がサポートを必要とするときにメンターシップとガイダンスを提供します。

#### 貢献者になる方法?

- StarRocksのパブリックリポジトリで1つのマージされたPR。

貢献者として、以下を期待しています:

- StarRocksのプロジェクト開発に積極的に参加すること。

- コミュニティイベント（ミートアップ、ハッカソンなど）に参加すること。

- 自分自身と他の人がStarRocks関連の技術を学び、助けること。

#### 特典

- StarRocksの貢献者としてリストアップされること。

- StarRocksの貢献者の電子証明書を授与されること。

### 積極的な貢献者

積極的な貢献者は、StarRocksへの優れた貢献と持続的なコミットを行っています。彼らは、コミュニティに積極的に参加し、コードを貢献し、ドキュメントを改善し、他の人を助けることによって活動しています。

#### 積極的な貢献者になる方法?

- マージされたPRが5つ以上または重要なバグが修正されたこと。

- 5つ以上のコードレビューに参加したこと。

- オンライン/オフラインのミートアップやコミュニティディスカッションなど、コミュニティイベントに積極的に参加したこと。

#### 責任と特権

- コミュニティのミーティングやディスカッションに参加すること。

- 新しい貢献者をメンターとしてサポートすること。

- StarRocks積極的な貢献者としてリストアップされること。

- StarRocks積極的な貢献者の電子証明書を授与されること。

### コミッター

コミッターは、積極的な貢献者から昇進します。

コミッターは、マスターブランチにPRをマージする権限を持ち、StarRocksの計画とメンテナンスに責任を持ちます。彼らはまた、コミュニティでの知識共有に積極的なメンバーでもあります。

#### コミッターになる方法?

- StarRocksの原則と将来の計画について深い理解を持つこと。

- プロジェクトにおいて発生するさまざまな問題に適切に対処する能力を持つこと。

- 主要な開発をリードし、関連する文書を作成および修正すること。

- 少なくとも2つのPMCノミネーションを受け取り、投票に合格すること。

#### 責任と特権

- コミュニティメンバーシップの他のメンバーをメンターとして指導すること。

- サブプロジェクトの継続的な健全性を確保すること。

- StarRocksリポジトリへの書き込みアクセス権が付与されること（指定される）。

- StarRocksコミッターとしてリストアップされること。

- StarRocksコミッターの電子証明書を授与されること。

### PMC

PMCは、コミッターから昇進します。マスターブランチにPRをマージする権限を持ち、StarRocksの計画とメンテナンスに責任を持ちます。彼らはまた、コミュニティでの知識共有に積極的なメンバーでもあります。

#### PMCになる方法?

- StarRocksの原則について深い理解とStarRocksの将来の計画について明確な理解を持つこと。

- プロジェクトにおいて発生する問題に対処する能力を持つこと。

- プロジェクトの開発とイテレーションをリードし、プロジェクトの全体的な方向性を指導すること。

- 少なくとも2つのPMCノミネーションを受け取り、投票に合格すること。

#### 責任と特権

- コミュニティメンバーシップの他のメンバーをメンターとして指導すること。

- コード品質やテストカバレッジなど、プロジェクトの健全性を維持すること。

- 技術的な設計の決定を行い、承認すること。

- マイルストーンとリリースを定義すること。

- 新しいコミッターやPMCの投票と昇進を行うこと。

- StarRocks PMCとしてリストアップされること。

- StarRocks PMCの電子証明書を授与されること。

## 貢献プロセス

始め方がわからない場合、以下の手順で貢献をすることをお勧めします。このプロセスは、学習コストを削減し、プルリクエストを効率よくマージするために設計されています！

1. [Contributor License Agreement (CLA)](https://cla-assistant.io/StarRocks/starrocks)に署名してください。

2. Github [issue](https://github.com/StarRocks/starrocks/issues)を作成するか、[Slackチャンネル](https://join.slack.com/t/starrocks/shared_invite/zt-1d0cs3cs2-7eL5Q0cGoBofhyIOx7pLfQ)で質問してください（変更がささいな場合を除く）。

   コードに手を付ける前に、問題の本文にコメントを残し、メンテナーに希望している問題を割り当ててもらうように伝えてください。また、問題の本文に問題を解決するための計画を共有することもお勧めします。

   - このステップは、可能な共同作業者とレビュアーを特定するのに役立ちます。
   - この変更は他の進行中の変更と競合しますか？もしそうなら、他の人と協力して影響を最小限に抑えてください。
   - この変更は大きな変更ですか？もしそうなら、他の人と協力して変更をより小さなステップに分割してください。

3. 変更を実装してください。

   - 変更が大規模な場合は、複数のPRに分割してください。

   - 必要に応じてテストとドキュメントを含めてください。
```
4. Create a Github [pull request](https://github.com/StarRocks/starrocks/pulls):

   In StarRocks community, we follow the fork-and-merge GitHub workflow when contributing code.

   - Create a fork of StarRocks in your GitHub account.
   - Clone this forked repository to your computer.
   - Check out a new branch based on the branch you expect to contribute to.
   - Commit your code changes to the new branch.
   - Push the branch with code changes to GitHub.
   - Create a PR to submit your code changes. It is recommended to submit ONE commit in ONE PR. You can follow the [PR templated](https://github.com/StarRocks/starrocks/blob/main/.github/PULL_REQUEST_TEMPLATE.md) when submitting a PR.
   - Make sure the pull request passes the tests in CI.

5. Review is required by at least 2 reviewers:

   - For StarRocks project, we require at least 2 lgtm from reviewers (committers) to merge the pull request.

   - This normally happens within a few days, but may take longer if the change is major, complex, or if a critical reviewer is unavailable. (feel free to ping the reviewer on the pull request).

6. Maintainers merge the pull request after the final changes are accepted.
```
