# cos_similarity

## 説明

2つのベクトルの類似性を計測し、それらの間の角度の余弦を計算して類似性を測定します。角度はベクトルの方向によって形成され、それらの大きさの差は無視されます。

類似性は-1から1の間です。ベクトル間の角度が小さいほど、余弦類似性は大きくなります。

- 2つのベクトルの方向が同じ場合、0度の角度を持ち、余弦類似性は1です。
- 直行するベクトルは90度の角度を持ち、余弦類似性は0です。
- 反対の方向を向くベクトルは180度の角度を持ち、余弦類似性は-1です。

`cosine_similarity` は、テキストやビデオの類似性の評価に一般的に使用されます。

この関数は、余弦類似性を測定する前にベクトルを正規化します。入力ベクトルが正規化されている場合、[cos_similarity_norm](./cos_similarity_norm.md) を使用できます。

## 構文

```Haskell
cosine_similarity(a, b)
```

## パラメータ

`a` と `b` は比較するベクトルです。それらは同じ次元を持っている必要があります。サポートされているデータ型は `Array<float>` です。2つの配列は同じ要素数を持っている必要があります。さもないと、エラーが返されます。

## 戻り値

[-1, 1] の範囲内の FLOAT 値を返します。入力パラメータが null または無効な場合、エラーが報告されます。

## 例

1. テーブルを作成し、このテーブルにデータを挿入するためのベクトルを格納するためのテーブルを作成します。

    ```SQL
    CREATE TABLE t1_similarity 
    (id int, data array<float>)
    DISTRIBUTED BY HASH(id);

    INSERT INTO t1_similarity VALUES
    (1, array<float>[0.1, 0.2, 0.3]), 
    (2, array<float>[0.2, 0.1, 0.3]), 
    (3, array<float>[0.3, 0.2, 0.1]);
    ```

2. `data` 列の各行の類似性を計算し、配列 `[0.1, 0.2, 0.3]` と比較して結果を降順でリストアップします。

    ```Plain
    SELECT id, data, cosine_similarity([0.1, 0.2, 0.3], data) as dist
    FROM t1_similarity 
    ORDER BY dist DESC;

    +------+---------------+-----------+
    | id   | data          | dist      |
    +------+---------------+-----------+
    |    1 | [0.1,0.2,0.3] | 0.9999999 |
    |    2 | [0.2,0.1,0.3] | 0.9285713 |
    |    3 | [0.3,0.2,0.1] | 0.7142856 |
    +------+---------------+-----------+
    ```
