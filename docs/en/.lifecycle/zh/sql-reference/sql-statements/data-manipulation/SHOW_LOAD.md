# 显示加载

## 描述

显示数据库中所有加载作业或给定加载作业的信息。此语句只能显示使用 [Broker Load](../data-manipulation/BROKER_LOAD.md)、[INSERT](./INSERT.md) 或 [Spark Load](../data-manipulation/SPARK_LOAD.md) 创建的加载作业。您还可以通过 `curl` 命令查看加载作业的信息。从 v3.1 开始，我们建议您使用 [SELECT](../data-manipulation/SELECT.md) 语句来查询 `information_schema` 数据库中的 `loads` 表，以获取 Broker Load 或 Insert 作业的结果。有关详细信息，请参阅从 [HDFS](../../../loading/hdfs_load.md) 加载数据、从云存储[加载数据、使用 INSERT 加载](../../../loading/cloud_storage_load.md)[数据](../../../loading/InsertInto.md)和[使用 Apache Spark™ 批量加载](../../../loading/SparkLoad.md)。

除了上述加载方式外，StarRocks 还支持使用 Stream Load 和 Routine Load 来加载数据。Stream Load 是同步操作，会直接返回 Stream Load 作业的信息。Routine Load 是异步操作，您可以使用 [SHOW ROUTINE LOAD](../data-manipulation/SHOW_ROUTINE_LOAD.md) 语句来显示 Routine Load 作业的信息。

## 语法

```SQL
SHOW LOAD [ FROM db_name ]
[
   WHERE [ LABEL { = "label_name" | LIKE "label_matcher" } ]
         [ [AND] STATE = { "PENDING" | "ETL" | "LOADING" | "FINISHED" | "CANCELLED" } ]
]
[ ORDER BY field_name [ ASC | DESC ] ]
[ LIMIT { [offset, ] limit | limit OFFSET offset } ]
```

> **注意**
>
> 您可以在语句中添加 `\G` 选项（例如 `SHOW LOAD WHERE LABEL = "label1"\G;`），以垂直显示输出，而不是通常的水平表格格式。有关详细信息，请参阅 [示例 1](#examples)。

## 参数

| **参数**                     | **必填** | **描述**                                              |
| --------------------------------- | ------------ | ------------------------------------------------------------ |
| db_name                           | 否           | 数据库名称。如果未指定此参数，则默认使用当前数据库。 |
| LABEL = "label_name"              | 否           | 加载作业的标签。                                     |
| LABEL LIKE "label_matcher"        | 否           | 如果指定此参数，则返回包含 `label_matcher` 的加载作业的信息。 |
| AND                               | 否           | <ul><li>如果在 WHERE 子句中只指定一个筛选条件，则不要指定此关键字。示例： `WHERE STATE = "PENDING"`.</li><li>如果在 WHERE 子句中指定了两个或三个筛选条件，则必须指定此关键字。示例： `WHERE LABEL = "label_name" AND STATE = "PENDING"`. </li></ul>|
| STATE                             | 否           | 加载作业的状态。状态根据加载方法而异。<ul><li>Broker Load<ul><li>`PENDING`：加载作业已创建。</li><li>`QUEUEING`：加载作业在队列中等待调度。</li><li>`LOADING`：加载作业正在运行。</li><li>`PREPARED`：事务已提交。</li><li>`FINISHED`：加载作业成功。</li><li>`CANCELLED`：加载作业失败。</li></ul></li><li>Spark Load<ul><li>`PENDING`：StarRocks 集群正在准备与 ETL 相关的配置，然后向您的 Apache Spark™ 集群提交 ETL 作业。</li><li>`ETL`：您的 Spark 集群正在执行 ETL 作业，然后将数据写入相应的 HDFS 集群。</li><li>`LOADING`：HDFS 集群中的数据正在加载到 StarRocks 集群中，表示加载作业正在运行。</li><li>`PREPARED`：事务已提交。</li><li>`FINISHED`：加载作业成功。</li><li>`CANCELLED`：加载作业失败。</li></ul></li><li>INSERT<ul><li>`FINISHED`：加载作业成功。</li><li>`CANCELLED`：加载作业失败。</li></ul></li></ul>如果未指定 `STATE` 参数，则默认返回所有状态下的加载作业信息。如果指定 `STATE` 参数，则仅返回给定状态下的加载作业信息。例如，`STATE = "PENDING"` 返回状态为 `PENDING` 的加载作业信息。 |
| ORDER BY field_name [ASC \| DESC] | 否           | 如果指定此参数，则根据字段按升序或降序对输出进行排序。支持以下字段：`JobId`、`Label`、`State`、`Progress`、`Type`、`EtlInfo`、`TaskInfo`、`ErrorMsg`、`CreateTime`、`EtlStartTime`、`EtlFinishTime`、`LoadStartTime`、`LoadFinishTime`、`URL` 和 `JobDetails`。<ul><li>要按升序对输出进行排序，请指定 `ORDER BY field_name ASC`。</li><li>要按降序对输出进行排序，请指定 `ORDER BY field_name DESC`。</li></ul>如果未指定字段和排序顺序，则默认按 `JobId` 升序排序。 |
| LIMIT limit                       | 否           | 允许显示的加载作业数。如果未指定此参数，则显示所有符合筛选条件的加载作业信息。如果指定此参数，例如 `LIMIT 10`，则仅返回符合筛选条件的 10 个加载作业的信息。 |
| OFFSET offset                     | 否           | `offset` 参数定义要跳过的加载作业数。例如，`OFFSET 5` 跳过前五个加载作业并返回其余的加载作业。`offset` 参数的值默认为 `0`。 |

## 使用说明

- SHOW LOAD 语句返回的信息在加载作业后的 3 天内有效 `LoadFinishTime` 。3 天后，信息无法显示。您可以使用该 `label_keep_max_second` 参数修改默认有效期。

    ```SQL
    ADMIN SET FRONTEND CONFIG ("label_keep_max_second" = "value");
    ```

- 如果 `LoadStartTime` 字段的值长时间为 `N/A`，则表示加载作业大量堆积。我们建议您降低创建加载作业的频率。
- 加载作业消耗的总时间段 = `LoadFinishTime` - `CreateTime`。
- 加载作业在 `LOAD` 阶段消耗的总时间段 = `LoadFinishTime` - `LoadStartTime`。
