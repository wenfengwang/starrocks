
StarRocks Stream Load 和 curl 支持多个参数。本教程中仅介绍了部分参数，其余参数将在更多信息部分提供链接。

#### `--location-trusted`

此配置使 curl 可以将凭证传递给任何重定向的 URL。

#### `-u root`

用于登录 StarRocks 的用户名。

#### `-T filename`

T 代表传输，即要传输的文件名。

#### `label:name-num`

与此 Stream Load 作业关联的标签。标签必须是唯一的，所以如果你多次运行作业，可以添加一个数字并递增该数字。

#### `column_separator:,`

如果你要加载的文件使用单个 `,` 作为分隔符，则按上述方式设置，如果使用其他分隔符，则在此设置。常见的选择有 `\t`、`,` 和 `|`。

#### `skip_header:1`

有些 CSV 文件的首行是标题行，列出了所有列名，有些文件可能会在第二行添加数据类型。如果你的文件有一到两行标题，请将 `skip_header` 设置为 `1` 或 `2`；如果没有标题行，则设置为 `0`。

#### `enclose:\"`

通常会用双引号将包含逗号的字符串括起来。本教程中使用的示例数据集中的地理位置包含逗号，因此 `enclose` 设置为 `\"`。记得用 `\` 转义 `"`。

#### `max_filter_ratio:1`

这允许数据中存在一些错误。理想情况下应该将其设置为 `0`，任何错误都会导致作业失败。这里设置为 `1` 是为了在调试时允许所有行失败。

#### `columns:`

CSV 文件列到 StarRocks 表列的映射。你会发现 CSV 文件中的列比表中的列要多。表中未包含的列将被忽略。

你还会注意到，对于崩溃数据集，`columns:` 行中包含了一些数据转换。在 CSV 文件中找到不符合标准的日期和时间是很常见的。以下是将崩溃日期和时间的 CSV 数据转换为 DATETIME 类型的逻辑：

##### 列行

这是一条数据记录的开始。日期格式为 `MM/DD/YYYY`，时间格式为 `HH:MI`。由于 DATETIME 通常格式为 `YYYY-MM-DD HH:MI:SS`，我们需要转换这些数据。

```plaintext
08/05/2014,9:10,BRONX,10469,40.8733019,-73.8536375,"(40.8733019, -73.8536375)",
```

这是 `columns:` 参数的开始：

```bash
-H "columns:tmp_CRASH_DATE, tmp_CRASH_TIME, CRASH_DATE=str_to_date(concat_ws(' ', tmp_CRASH_DATE, tmp_CRASH_TIME), '%m/%d/%Y %H:%i')
```

这指示 StarRocks 执行以下操作：
- 将 CSV 文件的第一列内容赋值给 `tmp_CRASH_DATE`
- 将 CSV 文件的第二列内容赋值给 `tmp_CRASH_TIME`
- `concat_ws()` 用空格将 `tmp_CRASH_DATE` 和 `tmp_CRASH_TIME` 连接起来
- `str_to_date()` 将连接后的字符串转换为 DATETIME
- 将得到的 DATETIME 存储在 `CRASH_DATE` 列中
