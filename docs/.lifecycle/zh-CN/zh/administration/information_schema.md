# 信息架构

`information_schema` is a database in the StarRocks instance. This database contains several tables defined by the system, which store a large amount of metadata information about all objects in the StarRocks instance.

Since v3.2, StarRocks supports viewing External Catalog metadata through `information_schema`.

## View Metadata Information through Information Schema

You can view the metadata information in the StarRocks instance by querying the tables in `information_schema`.

The following example queries the table `tables` to view the metadata information related to the table named `sr_member` in StarRocks.

```Plain
mysql> SELECT * FROM information_schema.tables WHERE TABLE_NAME like 'sr_member'\G
```

## Information Schema Tables

StarRocks optimizes the metadata information provided by the tables `tables`, `tables_config`, and `load_tracking_logs` in `information_schema`, and provides the table `loads` (since version 3.1):

| **Information Schema Table Name** | **Description**                                      |
| --------------------------- | ------------------------------------------------------------ |
| [tables](#tables)                      | Provides general table metadata information.                     |
| [tables_config](#tables_config)               | Provides additional metadata information unique to StarRocks.     |
| [load_tracking_logs](#load_tracking_logs)          | Provides error information related to import jobs.               |
| [loads](#loads)                       | Provides import job result information, supported since version 3.1. Currently, it only supports viewing the result information of [Broker Load](../sql-reference/sql-statements/data-manipulation/BROKER_LOAD.md) and [Insert](../sql-reference/sql-statements/data-manipulation/INSERT.md) import jobs.                  |

### loads

The table `loads` provides the following fields:

| **Field**             | **Description**                                                     |
| -------------------- | ------------------------------------------------------------ |
| JOB_ID               | The ID of the import job, generated by StarRocks.           |
| LABEL                | Label of the import job.                                       |
| DATABASE_NAME        | The name of the database where the target StarRocks table is located. |
| STATE                | The state of the import job, including: `<ul><li>`PENDING`：The import job has been created.</li><li>`QUEUEING`：The import job is waiting for execution.</li><li>`LOADING`：The import job is in progress.</li><li>`PREPARED`：The transaction has been committed.</li><li>`FINISHED`：The import job is successful.</li><li>`CANCELLED`：The import job has failed.</li></ul>See [Asynchronous Import](../loading/Loading_intro.md#Asynchronous_ Import). |
| PROGRESS             | The progress of the ETL and LOADING stages of the import job.  |
| TYPE                 | The type of the import job. If it is a Broker Load import, it returns `BROKER`. If it is an INSERT import, it returns `INSERT`. |
| PRIORITY             | The priority of the import job. Possible values: `HIGHEST`, `HIGH`, `NORMAL`, `LOW`, `LOWEST`. |
| SCAN_ROWS            | Total number of scanned data rows.                            |
| FILTERED_ROWS        | Total number of error data rows filtered due to data quality issues. |
| UNSELECTED_ROWS      | Total number of data rows filtered based on the conditions specified in the WHERE clause. |
| SINK_ROWS            | Total number of data rows imported.                           |
| ETL_INFO             | ETL details. This field is only valid for Spark Load jobs. If it is another type of import job, the result is empty. |
| TASK_INFO            | Import job details, such as timeout duration (`timeout`) and maximum tolerance ratio (`max_filter_ratio`). |
| CREATE_TIME          | The time when the import job was created. Format: `yyyy-MM-dd HH:mm:ss`. For example, `2023-07-24 14:58:58`. |
| ETL_START_TIME       | Start time of the ETL stage. Format: `yyyy-MM-dd HH:mm:ss`. For example, `2023-07-24 14:58:58`. |
| ETL_FINISH_TIME      | End time of the ETL stage. Format: `yyyy-MM-dd HH:mm:ss`. For example, `2023-07-24 14:58:58`. |
| LOAD_START_TIME      | Start time of the LOADING stage. Format: `yyyy-MM-dd HH:mm:ss`. For example, `2023-07-24 14:58:58`. |
| LOAD_FINISH_TIME     | End time of the LOADING stage. Format: `yyyy-MM-dd HH:mm:ss`. For example, `2023-07-24 14:58:58`. |
| JOB_DETAILS          | Import data details, including the size of imported data (in bytes), number of files, etc. |
| ERROR_MSG            | Error information for the import job. If there are no errors, it returns `NULL`. |
| TRACKING_URL         | Access address of the poor-quality data sample in the import job. You can use the `curl` command or `wget` command to access this address. If there is no poor-quality data in the import job, it returns `NULL`. |
| TRACKING_SQL         | Query statement for import job tracking logs. It is only returned when the import job contains poor-quality data. If the import job does not contain poor-quality data, it returns `NULL`. |
| REJECTED_RECORD_PATH | Access address of poor-quality data in the import job. The number of poor-quality data results returned is determined by the `log_rejected_record_num` parameter in the import job. You can use the `wget` command to access this address. It returns `NULL` if there is no poor-quality data in the import job. |

### tables

The table `tables` provides the following fields:

| **Field**        | **Description**                                                     |
| --------------- | ------------------------------------------------------------ |
| TABLE_CATALOG   | The name of the Catalog to which the table belongs.            |
| TABLE_SCHEMA    | The name of the database to which the table belongs.           |
| TABLE_NAME      | The name of the table.                                          |
| TABLE_TYPE      | The type of the table. Valid values: "BASE TABLE" or "VIEW".   |
| ENGINE          | The type of the table's engine. Valid values: "StarRocks", "MySQL", "MEMORY", or an empty string. |
| VERSION         | This field is currently not available.                         |
| ROW_FORMAT      | This field is currently not available.                         |
| TABLE_ROWS      | The number of rows in the table.                               |
| AVG_ROW_LENGTH  | The average row length of the table, equal to `DATA_LENGTH` / `TABLE_ROWS`. Unit: byte. |
| DATA_LENGTH     | The length of the table's data file. Unit: byte.               |
| MAX_DATA_LENGTH | This field is currently not available.                         |
| INDEX_LENGTH    | This field is currently not available.                         |
| DATA_FREE       | This field is currently not available.                         |
| AUTO_INCREMENT  | This field is currently not available.                         |
| CREATE_TIME     | The time when the table was created.                           |
| UPDATE_TIME     | The time of the last update to the table.                      |
| CHECK_TIME      | The time of the last consistency check of the table.           |
| TABLE_COLLATION | The default collation of the table.                            |
| CHECKSUM        | This field is currently not available.                         |
| CREATE_OPTIONS  | This field is currently not available.                         |
| TABLE_COMMENT   | 表的注释。                                               |

### tables_config

表 `tables_config` 提供以下字段：

| **字段**         | **描述**                                                     |
| ---------------- | ------------------------------------------------------------ |
| TABLE_SCHEMA     | 表所属的数据库名称。                                         |
| TABLE_NAME       | 表名。                                                       |
| TABLE_ENGINE     | 表的引擎类型。                                               |
| TABLE_MODEL      | 表的数据模型。 有效值：“DUP_KEYS”、“AGG_KEYS”、“UNQ_KEYS” 或 “PRI_KEYS”。 |
| PRIMARY_KEY      | 主键模型或更新模型表的主键。如果该表不是主键模型或更新模型表，则返回空字符串。 |
| PARTITION_KEY    | 表的分区键。                                                 |
| DISTRIBUTE_KEY   | 表的分桶键。                                                 |
| DISTRIBUTE_TYPE  | 表的分桶方式。                                               |
| DISTRIBUTE_BUCKET | 表的分桶数。                                                 |
| SORT_KEY         | 表的排序键。                                                 |
| PROPERTIES       | 表的属性。                                                   |
| TABLE_ID         | 表的 ID。                                                   |

### load_tracking_logs

此功能自 StarRocks v3.0 起支持。

表 `load_tracking_logs` 提供以下字段：

| **字段**         | **描述**                                                     |
| ---------------- | ----------------------------------------------------------- |
| JOB_ID           | 导入作业的 ID。                                               |
| LABEL            | 导入作业的 Label。                                            |
| DATABASE_NAME    | 导入作业所属的数据库名称。                                      |
| TRACKING_LOG     | 导入作业的错误日志信息（如有）。                                 |

| INACTIVE_REASON                      | 失效的原因                                       |
| PARTITION_TYPE                       | 物化视图分区类型                                 |
| TASK_ID                              | 物化视图刷新任务的 ID                            |
| TASK_NAME                            | 物化视图刷新任务的名称                           |

| LAST_REFRESH_FINISHED_TIME           | 最近一次刷新任务的结束时间                       |
| LAST_REFRESH_DURATION                | 最近一次刷新任务的持续时间                       |
| LAST_REFRESH_STATE                   | 最近一次刷新任务的状态                           |
| LAST_REFRESH_FORCE_REFRESH           | 最近一次刷新任务是否强制刷新                     |
| LAST_REFRESH_START_PARTITION         | 最近一次刷新任务的开始分区                       |
| LAST_REFRESH_END_PARTITION           | 最近一次刷新任务的结束分区                       |
| LAST_REFRESH_BASE_REFRESH_PARTITIONS | 最近一次刷新任务的基表分区                       |
| LAST_REFRESH_MV_REFRESH_PARTITIONS   | 最近一次刷新任务的刷新的分区                     |

| LAST_REFRESH_ERROR_MESSAGE           | 最近一次刷新任务的错误信息                       |
| TABLE_ROWS                           | 物化视图的数据行数，后台统计的近似值             |
| MATERIALIZED_VIEW_DEFINITION         | 物化视图的 SQL 定义                              |
